Package: edu.holycross.shot.hocuspocus

[Groovy] Class TextInventory

    • edu.holycross.shot.hocuspocus.TextInventory

    • Class modelling a TextInventory. Methods for finding artifacts in the TextInventory appear in two forms:

      • an internal "private" method returning a TextInventory class data structure. These methods have 'Data' in their names.
      • a public method returning URN values or String values
      The internal 'Data' methods should not be used: internal data structures may change. The URN and String methods are stable. They will not change their signature and should produce identical results even if the underlying implementation changes.

      • Property Detail

        • java.lang.Object ctsnamespaces

          A list of namespace triplets. Each CTS namespace is comprised of a namespace abbreviation, a full URI, and a description. These can be read from an XML TextInventory file.

        • static groovy.xml.Namespace ctsns

          XML namespace for the CTS vocabulary.

        • java.lang.Object editions

          A list of version-level structures. Each version-level structure is comprised of a CTS URN for the version, a label, a boolean flag indicating whether or not this version is online, and a parent (work) URN. These can be read from an XML TextInventory file.

        • java.lang.String enc

          Character encoding to use when representing TextInventory as a File object.

        • java.lang.Object errorList

          List of data values that violate the TextInventory definition.

        • java.lang.Object exemplars

          A list of exemplar-level structures. Each exemplar-level structure is comprised of a CTS URN for the exemplar, a label, a boolean flag indicating whether or not this version is online, and a parent (version-level) URN. These can be read from an XML TextInventory file.

        • java.lang.Object textgroups

          A list of textgroup structures. Each textgroup structure is comprised of a CTS urn and a label. Although the XML schema for a TextInventory allows multiple labels for a textgroup, this data structure keeps only one. These can be read from an XML TextInventory file.

        • static groovy.xml.Namespace ti

          XML namespace for the TextInventory vocabulary.

        • java.lang.Object tiVersion

          The TextInventory ID. In an XML serialization validating against the TextInventory schema, this is the tiid attribute on the root element.

        • java.lang.Object translationLanguages

          Map of version-level Cts Urns to ISO three-letter language codes. This map can be read from an XML TextInventory file.

        • java.lang.Object translations

          A list of version-level structures. Each version-level structure is comprised of a CTS URN for the version, a label, a boolean flag indicating whether or not this version is online, and a parent (work) URN. These can be read from an XML TextInventory file.

        • java.lang.Object works

          A list of work structures. Each work structure is comprised of a CTS URN, a title, and a parent (textgroup) URN. (A full work-level URN can therefore be composed by joining the work-level URN and the work ID). Although the XML schema for a TextInventory allows multiple titles for a work, this data structure keeps only one. These can be read from an XML TextInventory file.

        • java.lang.Object worksLanguages

          Map of work-level Cts Urns to ISO three-letter language codes. This map can be read from an XML TextInventory file.
